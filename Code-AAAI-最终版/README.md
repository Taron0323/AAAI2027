# ForeAct Code Repository - Final

This final repository integrates `Code-AAAI` and `Code-AAAI-V2版` for **ForeAct: Latent Future-Action Prediction and Plan-Consistency Regularization for Long-Horizon LLM Agents**.
It follows `ForeAct_论文整体框架_v3模板填充版.md` as the highest-priority product contract and cross-checks the current AAAI draft in `AuthorKit27/ForeAct_AAAI2027.pdf`.

The code is organized around the paper phases:

- **Phase 1:** trajectory parsing, PlanDepth generation, action-sketch `zeta`, and K-rollout soft targets.
- **Phase 2:** Latent Future-Action Prediction (LAP) heads and future-sketch training losses.
- **Phase 3:** Plan-Consistency Regularization (PCR) with the forward KL direction required by Proposition 1.
- **Phase 4:** zero-overhead ReAct-style inference plus optional latent reranking.

No headline paper results are generated by the smoke path. The included commands only verify that the engineering path runs end to end on tiny synthetic data.

This directory intentionally reuses the already-downloaded local official assets through symlinks:

- `third_party/benchmarks` -> `../Code-AAAI/third_party/benchmarks`
- `third_party/baselines` -> `../Code-AAAI/third_party/baselines`
- `third_party/auxiliary` -> `../Code-AAAI/third_party/auxiliary`
- `datasets` -> `../Code-AAAI/datasets`

The symlink design avoids duplicating gigabytes of benchmark/baseline code and datasets while preserving local availability checks.

## V3 Paper Contract

The current code follows `ForeAct_论文整体框架_v3模板填充版.md` as the active product contract:

- North-star claim: **Plan like diffusion, act like autoregressive.**
- The action-granularity contrast is protected by the token-level MTP baseline, A' sketch-granularity jobs, and Remark-1 degenerate-corner registry entries.
- PCR uses `KL(sg[q_{t+1}^{h-1}] || q_t^h)` and exposes no-PCR / no-branch-weight controls.
- Mode A remains ReAct-shaped: no special prompt tokens, no forecast decoding, no extra inference tokens.
- G anti-confounds include predict-past and FLOPs-matched SFT controls.
- Smoke artifacts are explicitly labeled and must not be copied into the paper as measured results.

## Real Experiment Hooks

The repository intentionally separates smoke-safe code from full-scale dependencies:

- `foreact.models.foreact_torch` contains a PyTorch implementation that works when `torch` is installed.
- `foreact.models.hf_backbone` contains optional Hugging Face/Qwen action-state extraction helpers.
- `foreact.teachers` defines provider abstractions for DeepSeek-V4-Pro, DeepSeek-V4-Flash, GPT-style second teachers, and mock teachers.
- `foreact.data.benchmarks` and `foreact.data.prepare` convert official tau2/AppWorld/SWE assets into ForeAct data contracts.
- `foreact.evaluation.adapters` contains harness adapters for tau2-bench, AppWorld, SWE-Gym, and SWE-bench Verified. If a benchmark is not installed locally, adapters return a skipped status with setup guidance instead of failing mysteriously.
- `foreact.evaluation.runners` and `foreact.baselines.runners` build official harness/baseline commands without pretending dry-run plans are measured results.

Secrets are never hard-coded. API providers read keys from environment variables and support cache/mocking.
