# ForeAct Code Repository

This repository implements the pre-experiment code path for **ForeAct: Latent Future-Action Prediction and Plan-Consistency Regularization for Long-Horizon LLM Agents**.

The code is organized around the paper phases:

- **Phase 1:** trajectory parsing, PlanDepth generation, action-sketch `zeta`, and K-rollout soft targets.
- **Phase 2:** Latent Future-Action Prediction (LAP) heads and future-sketch training losses.
- **Phase 3:** Plan-Consistency Regularization (PCR) with the forward KL direction required by Proposition 1.
- **Phase 4:** zero-overhead ReAct-style inference plus optional latent reranking.

No headline paper results are generated by the smoke path. The smoke command only verifies that the engineering path runs end to end on tiny synthetic data; the real entrypoints below prepare official datasets, launch HF/Qwen training, materialize ablations, and build official benchmark harness commands.

## Quick Start

```bash
cd /root/autodl-tmp/foceact/Code-AAAI
python3 -m pip install -e ".[dev]"
python3 -m pytest
python3 -m foreact.cli smoke --config configs/smoke.yaml
```

The smoke command writes small artifacts under `outputs/smoke/`:

- PlanDepth trajectories
- generated sketch schema
- aligned LAP/PCR training examples
- toy training metrics
- ReAct and latent-rerank evaluation summaries
- CSV files ready for paper plots
- smoke-shaped ELD and forecast-entropy diagnostics
- dependency-free SVG summaries and a PlanDepth case-study heatmap

Useful registry/setup commands:

```bash
python3 -m foreact.cli registry --out outputs/smoke/registry.json
python3 -m foreact.cli ablation-jobs --config configs/ablation_matrix.yaml --out outputs/smoke/ablation_jobs.json
python3 -m foreact.cli harness-status --out outputs/smoke/harness_status.json
python3 -m foreact.cli asset-status --out outputs/smoke/asset_status.json
```

Real pre-experiment commands:

```bash
PYTHONPATH=src python3 -m foreact.cli prepare-data --config configs/foreact_4b_milestone.yaml
PYTHONPATH=src python3 -m foreact.cli train-foreact --config configs/foreact_4b_milestone.yaml
PYTHONPATH=src python3 -m foreact.cli probe-eld --config configs/foreact_4b_milestone.yaml

PYTHONPATH=src python3 -m foreact.cli generate-trajectories --config configs/teacher_trajectories.yaml
PYTHONPATH=src python3 -m foreact.cli run-baseline --config configs/baseline_lats.yaml
PYTHONPATH=src python3 -m foreact.cli run-efficiency --config configs/efficiency_vllm.yaml
PYTHONPATH=src python3 -m foreact.cli build-control-task --config configs/control_task.yaml
```

Official benchmark harnesses can be checked and command-planned with:

```bash
bash scripts/install_external_packages.sh
bash scripts/create_appworld_env.sh
PYTHONPATH=src python3 -m foreact.cli run-benchmark --config configs/benchmark_swe_verified.yaml
```

To fetch official external benchmark and baseline code into `third_party/`, run:

```bash
bash scripts/bootstrap_external_assets.sh
```

If local git access to GitHub is unstable, retry with archive downloads:

```bash
bash scripts/bootstrap_external_assets.sh archive
```

Official datasets are not committed. Create/check their local targets with:

```bash
bash scripts/download_datasets.sh
```

## Real Experiment Hooks

The repository intentionally separates smoke-safe code from full-scale dependencies:

- `foreact.models.foreact_torch` contains a PyTorch implementation that works when `torch` is installed.
- `foreact.models.hf_backbone` contains optional Hugging Face/Qwen action-state extraction helpers.
- `foreact.teachers` defines mock and OpenAI-compatible cached API providers for DeepSeek-V4-Pro, DeepSeek-V4-Flash, and GPT-style second teachers.
- `foreact.data.benchmarks` and `foreact.data.prepare` convert official tau2/AppWorld/SWE-Gym/SWE-bench assets into ForeAct trajectory, evaluation, schema, and aligned-example artifacts.
- `foreact.evaluation.runners` builds official tau2/AppWorld/SWE-bench/mini-SWE-agent harness commands and can execute them when the local environment is installed.

Secrets are never hard-coded. API providers read keys from environment variables and support cache/mocking.

See `docs/FULL_TRAINING.md` for the Qwen/HF trainer contract.
See `docs/COMPLETION_AUDIT.md` for requirement-to-file evidence and remaining external dependencies.
See `docs/DATASETS_AND_BASELINES.md` for the distinction between built-in smoke code, external adapters, and real official data/baselines.
